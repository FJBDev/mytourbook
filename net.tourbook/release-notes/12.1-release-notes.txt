MyTourbook 12.1 BETA 1 release notes (?? December 2011)


!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
                       
		       VERY IMPORTANT INFO

This version has major structural changes in the tour database. 
It is STRONGLY recommended to do a backup of the data before the 
new version is started.

!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!



Major Improvements (see details below):

- modernized tour chart, behaviour is different than before
- value point tooltip in the tour chart
- all data series are saved in floating point format
- altitude values are displayed exactly as saved in the import file, the altitude values for already saved tours can be reimported but adjusted altitude values can not be preserved


Other Improvements:

- in the adjust altitude dialog with the method "approach altitude with SRTM" a new checkbox is available to select the whole tour
- reimporting a tour can also be done for altitude values only
- reimporting can also be done in the tourbook view (available in the context menu)


Tour Chart Improvements:

- transparency and antialiasing for the line graphs
- major and minor grid lines 
- improved performance for drawing the tour chart
- increased chart virtual size to 10'000'000 pixel: this is factor 100 more than before
	

New Tour Chart Behaviour:

- when the chart is zoomed with the mouse wheel, the horizontal position in the chart will be preserved where the mouse is located (in the previous versions the last mouse double click position was preserved, this is still the behaviour of the map)
- scroll tour chart when vertical sliders are dragged outside of the visible area (for a zoomed chart)
- when the chart is zoomed in and the mouse is moved within the x-axis area, the graph is scrolled left/right
- Ctrl+Shift moves the x-slider to the exact value position (Ctrl MUST be pressed FIRST!!!)
- mouse wheel zoom in the vertical axis is zooming the left and right border
- the mouse has 3 different vertical positions in the chart which are displayed with different cursors:
  - upper part: when the mouse is clicked, the left moust slider will be positions to the mouse
  - lower part: when the mouse is clicked, the right moust slider will be positions to the mouse
  - middle part: when the mouse is clicked and button is kept down, the chart can be dragged (this is the behaviour in previous versions)
- when the mouse is clicked on a vertical slider, this slider is pinned to the mouse until a second mouse click occures
- horizontal scrolling a zoomed chart with the scrollbar is removed, this was available in the statistic charts


Hints for using Tour Chart and Map:
- when a vertical slider in the tour chart is not moving the slider in the map, ensure that the tourchart has the focus


Value Point Tooltip

- this tooltip has many features which can be set with the button in the top/right corner of the tooltip
- when the tooltip has been set to be hidden, the chart options menu has the menu item "Show Value Point Tooltip"
- the tooltip can be dragged with the mouse, the selected pin position keeps the tooltip when the chart is move or resized


Update Infos:

-conversion time: 1727 tours / 149 seconds = 11.6 tours/second


System:
- Eclipse 3.7.1
- Derby 10.8.2.1
-decreased number of plugins, moved most of the device plugins into net.tourbook.device


Fixed bugs:

- Tour editor not dirty when changing tour type https://sourceforge.net/tracker/?func=detail&aid=3459317&group_id=179799&atid=890601
- a tour with 53952 time slices cannot be saved https://sourceforge.net/tracker/?func=detail&aid=3421203&group_id=179799&atid=890601
